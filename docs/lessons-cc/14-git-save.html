Git: сохранение работы

Git — это система сохранений. Как в видеоиграх: сохранился, пошёл экспериментировать, если что-то сломалось — вернулся на чекпоинт.

Зачем

Представь: что-то сделано, потом ещё, Claude что-то сломал, ты не заметил, сделал ещё... И ничего не работает.

Без Git: сидишь и вспоминаешь что менялось. Или начинаешь заново.

С Git: откатываешься к последнему рабочему сохранению. 30 секунд — и всё работает.

Два главных слова

  • Commit (коммит) — сохранить точку

  • Push (пуш) — отправить сохранения на GitHub

💡 Простыми словами: Commit = сохранился в игре (на своём компе). Push = загрузил сохранение в облако (GitHub).

Когда делать commit

Правило: сделал что-то рабочее — сохранись.

  • Добавил новую страницу и она работает → commit

  • Починил ошибку → commit

  • Закончил работу на сегодня → commit

Маленькие частые коммиты лучше чем один большой.

Как делать commit

Сделай коммит с сообщением "Добавил страницу контактов"

Или:

Сохрани текущие изменения в git

Когда делать push

  • Сделал 2–3 коммита → push

  • Закончил работу → push

  • Хочешь бэкап в облаке → push

Отправь изменения на GitHub

Полный цикл

Поработал → Commit → Поработал → Commit → Push

Пример дня:

  1. Утро: добавил форму → commit «Добавил форму»

  2. День: починил баг → commit «Исправил ошибку»

  3. Вечер: добавил стили → commit «Добавил стили»

  4. Уходишь → push

Как вернуться назад

Покажи историю коммитов

Верни проект к коммиту "Добавил форму"

⚠️ Осторожно: Возврат отменяет все изменения после этого коммита.

Типичные ошибки

  1. Забыл коммит — нечего откатывать. Решение: коммить чаще.

  2. Забыл push — коммиты есть, но только на компе. Решение: push в конце работы.

  3. Коммит с нерабочим кодом — откат не поможет. Решение: коммить когда работает.

Практика

  1. Сделай любое маленькое изменение

  2. Попроси: Сделай коммит «Тестовый коммит»

  3. Попроси: Push

  4. Открой github.com — там должен появиться коммит

Тест

2. Что делает push?

3. Как часто нужно коммитить?
