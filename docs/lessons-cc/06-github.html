Подключение GitHub

GitHub — это место где лежит твой код в интернете. Он хранит историю всех изменений, даёт возможность вернуться к любой версии и позволяет показывать код другим. Без него никак не обойтись.

Словарь

  • Git — программа на компьютере, которая отслеживает изменения в файлах. Запоминает каждое сохранение.

  • GitHub — сайт, куда загружаешь проекты. Как облачное хранилище, но умнее — понимает историю изменений.

  • Репозиторий (repo) — один проект на GitHub. Одна папка = один репозиторий.

Зачем тебе GitHub

1. Бэкап в облаке.

Ноутбук сломался? Кофе на клавиатуру? Если проект на GitHub — ничего не потеряно. Скачаешь на новую машину и продолжишь.

2. История изменений.

GitHub запоминает каждое сохранение. Сделал что-то не то сегодня — можно вернуться к версии недельной или месячной давности.

3. Портфолио.

Когда начнёшь делать реальные проекты, GitHub станет портфолио. Можно показать работодателю или клиенту.

💡 Простыми словами: GitHub — как Google Диск для кода. Только умнее: понимает что изменилось и хранит всю историю.

Создание аккаунта

Если уже есть аккаунт — пропусти этот раздел.

  1. Открой github.com

  2. Нажми Sign Up

  3. Введи email, придумай пароль, выбери username

  4. Подтверди email (проверь почту)

Username — твой никнейм. Он будет в ссылках на проекты. Выбирай что-то нормальное.

Подключение VS Code к GitHub

Способ 1 — через Command Palette:

  1. Открой VS Code

  2. Cmd/Ctrl + Shift + P → напиши GitHub: Sign In

  3. Откроется браузер — войди в GitHub и разреши доступ

  4. Вернись в VS Code

Способ 2 — через иконку аккаунта:

  1. Нажми на иконку человечка внизу слева

  2. Выбери Sign in with GitHub

  3. Разреши доступ в браузере

⚠️ Если авторизация не работает: 1. Возможно нужен Git — попроси Claude: «Установи Git» 2. Попробуй другой браузер 3. Выйди из GitHub в браузере и войди заново

Создание репозитория

  1. Открой github.com и войди в аккаунт

  2. Нажми зелёную кнопку New (или + → New repository)

  3. Введи название (например: my-first-project)

  4. Выбери Public или Private

  5. Важно: НЕ ставь галочки «Add a README file» — оставь пустым

  6. Нажми Create repository

💡 Public vs Private:

  • Private — только ты видишь код. Безопасно для экспериментов.

  • Public — любой может увидеть код. Используй когда хочешь показать проект миру.

Рекомендую: всегда выбирай Private. Если понадобится показать — можно изменить потом.

Подключение к проекту

Если папка пустая (новый проект):

  1. Cmd/Ctrl + Shift + P → Git: Clone

  2. Вставь URL репозитория

  3. Выбери папку куда сохранить

Если уже есть файлы:

  1. Открой проект в VS Code

  2. Cmd/Ctrl + Shift + P → Publish to GitHub

  3. Выбери Public или Private

  4. Файлы загрузятся на GitHub

Проверка

  1. Открой github.com

  2. Зайди в профиль → Repositories

  3. Там должен быть твой репозиторий

Как это будет работать

Дальше по курсу:

  1. Делаешь изменения — Claude создаёт/редактирует файлы

  2. Сохраняешь локально — Accept изменения

  3. Отправляешь на GitHub — это называется «push»
