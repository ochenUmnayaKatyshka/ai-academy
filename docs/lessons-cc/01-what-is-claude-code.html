Что такое Claude Code и зачем он нужен

Представь: у тебя в голове есть проект. Сайт, бот, приложение, автоматизация — что угодно. Раньше между идеей и реализацией стояла стена: нужно было либо учиться программировать (месяцы), либо нанимать разработчика (деньги), либо забить и двигаться дальше без этого.

В 2025 году эта стена рухнула. И Claude Code — один из инструментов, который её снёс.

Что это такое — простым языком

Claude Code — это AI-помощник от компании Anthropic. Он работает внутри специальной программы на твоём компьютере (мы установим её в следующем уроке — ничего сложного). Он не просто отвечает на вопросы в чате — он видит твои файлы, может их менять, создавать новые, удалять ненужные и запускать команды на твоём компьютере.

Ты пишешь запрос: «сделай мне страницу с формой обратной связи». Он находит нужные файлы, вносит изменения, сохраняет — и ты видишь результат. Без копирования кода из браузера. Без вопроса «а куда это вставить».

На каком языке с ним разговаривать?

Честный ответ: Claude обучался в основном на английском. Это значит, что на английском он понимает точнее, формулирует чётче и реже ошибается.

Но это не значит, что на русском он работает плохо. Для задач, которые мы будем делать в этом курсе, русского более чем достаточно. Claude отлично понимает русский язык — ты можешь писать ему обычными словами, и он будет делать то, что нужно.

💡 Про язык — не переживай: На старте пиши на русском — так проще и понятнее. Чуть позже в курсе мы разберём, как формулировать запросы на английском, даже если ты его не знаешь (спойлер: Claude сам поможет с переводом). Но для первых проектов русский — идеальный выбор.

Почему именно Claude Code, а не ChatGPT

Этот вопрос задают все. Давай разберёмся честно.

Обычные AI-чаты (ChatGPT, Claude.ai, Gemini)

Это разговорные инструменты. Ты задаёшь вопрос — получаешь ответ в виде текста. Если тебе нужен код — он покажет его в окне чата. Дальше ты: - Копируешь код - Создаёшь файл - Вставляешь туда код - Если что-то не работает — копируешь ошибку обратно в чат - Получаешь новый код, опять копируешь...

Это работает. Но это медленно и утомительно, особенно когда проект разрастается до десятков файлов.

Claude Code

Это агентный инструмент. Слово «агентный» означает, что он не просто отвечает — он действует. У него есть доступ к твоим файлам и возможность выполнять действия:

  • Прочитать любой файл в проекте
  • Создать новый файл
  • Изменить существующий код
  • Удалить файл
  • Запустить команду в терминале (например, установить библиотеку)
  • Посмотреть результат и исправить ошибки

И всё это — внутри одного окна. Без переключений, без копирования, без вопросов «а куда это вставить».

💭 Комментарий от Катеньки: Я не призываю бросить ChatGPT или любой другой инструмент. Каждый хорош для своего. ChatGPT отлично объясняет, гуглит, рассуждает. Но когда дело доходит до «сделай мне проект» — Claude Code на другом уровне.

Как это устроено технически (без паники)

Не нужно запоминать это наизусть. Но полезно понимать как инструмент работает, чтобы потом не удивляться его поведению.

Claude Code работает локально на твоём компьютере. Это ключевое отличие от обычного чата. Когда ты открываешь Claude Code в VS Code:

  1. Он получает доступ к папке твоего проекта
  2. Может читать все файлы внутри этой папки
  3. Перед каждым изменением показывает тебе что он хочет изменить
  4. Ждёт твоего подтверждения (или работает в автоматическом режиме — об этом позже)
  5. Вносит изменения только после твоего «ок»

Контекстное окно — это сколько информации Claude может держать «в голове» одновременно. У Claude Code это около 200 000 токенов. Грубо говоря — это как прочитать небольшую книгу за раз. Когда проект большой и информации становится слишком много, Claude сам «сжимает» разговор — убирает неважное, оставляет суть, и продолжает работу.

Модели. За кулисами Claude Code использует разные AI-модели от Anthropic: - Opus — самая умная, медленная и дорогая. Для сложных задач. - Sonnet — золотая середина. Быстрая и достаточно умная. Это основная рабочая лошадка. - Haiku — самая быстрая и дешёвая. Для простых задач.

Ты не выбираешь модель вручную (хотя можешь). По умолчанию Claude Code сам решает что использовать.

Что конкретно можно делать

Не абстрактно, а конкретно — примеры реальных задач:

Создать с нуля

  • Лендинг или многостраничный сайт
  • Телеграм-бот (уведомления, автоответы, приём заказов)
  • Веб-приложение (калькулятор, таск-менеджер, CRM)
  • Скрипт для автоматизации (парсинг данных, отправка писем, обработка файлов)

Работать с чужим кодом

  • Разобраться в проекте, который ты видишь впервые — Claude прочитает все файлы и объяснит что к чему
  • Найти и исправить ошибку
  • Добавить новую функцию в существующий проект
  • Переписать кусок кода, который работает, но выглядит как кошмар

Задачи, где кодинг — не главное

  • Написать техническую документацию
  • Составить ТЗ (техническое задание) для проекта
  • Проанализировать данные из CSV/Excel
  • Настроить деплой (публикацию проекта в интернет)

💡 Важный принцип: Всё что можно описать словами — Claude может попробовать сделать. Он не всегда сделает идеально с первого раза. Но направление задаёшь ты, а черновую работу делает он.

А если я не программист?

Весь смысл Claude Code — дать возможность создавать проекты без знания языков программирования. Это не маркетинговый ход — это реально работает.

Что нужно для старта: - Уметь объяснить что ты хочешь (обычными словами, по-русски) - Быть готовым к тому, что с первого раза будет не идеально - Не бояться пробовать и экспериментировать

Что не нужно: - Знать Python, JavaScript или любой другой язык - Понимать что такое «переменная», «функция» или «класс» - Иметь техническое образование

Со временем ты неизбежно начнёшь понимать код — просто от того, что будешь видеть его каждый день. Это как с иностранным языком: если живёшь в среде — начинаешь понимать без учебников.

💭 Комментарий от Катеньки: Я создавала этот курс для тех, кто никогда не программировал. Все инструкции написаны так, чтобы можно было повторить каждый шаг. Если где-то непонятно — это проблема инструкции, не ваша.

Сравнение с другими инструментами

На рынке есть несколько AI-инструментов для работы с кодом. Вот честное сравнение:

Инструмент Тип Плюсы Минусы
ChatGPT Чат Хорошо объясняет, широкие знания Не работает с файлами напрямую, нужно копировать код вручную
GitHub Copilot Автодополнение Быстрые подсказки прямо в коде Подсказывает по строчке, не понимает проект целиком
Cursor Редактор с AI Мощный, работает с файлами Отдельный редактор, к которому надо привыкать
Claude Code Агент в VS Code Видит весь проект, сам вносит изменения, работает в знакомой среде Платный, зависит от интернета

Каждый инструмент хорош для своего. Мы используем Claude Code, потому что он самый популярный и лучше всего подходит для нашей задачи: создание проектов с нуля силами человека без опыта в коде.

Честные ограничения

Было бы нечестно говорить только о плюсах. Вот что нужно знать:

  • Интернет обязателен. Claude Code отправляет твои запросы на серверы Anthropic. Без интернета он не работает.
  • Он ошибается. Иногда генерирует код с багами. Иногда неправильно понимает задачу. Это нормально — ты учишься с ним итеративно (попросить → проверить → поправить).
  • Большие проекты = больше сложностей. На маленьком лендинге Claude работает как волшебник. На проекте с сотней файлов — нужно больше структуры и дисциплины.
  • Конфиденциальность. Твой код отправляется на серверы Anthropic для обработки. Для учебных проектов это не проблема, но для работы с секретными данными — имей в виду.
