# Как разговаривать с Claude

Claude понимает обычный язык. Но есть разница между «понимает» и «делает именно то, что ты хочешь». Разберёмся как получать нужный результат.

---

## Главное правило

Чем конкретнее запрос — тем лучше результат.

**Размыто:**
```
Сделай страницу
```
Claude: «Какую? О чём? Какой дизайн?»

**Конкретно:**
```
Сделай страницу с заголовком "Мой блог" и тремя абзацами текста
```

**Ещё конкретнее:**
```
Сделай страницу с заголовком "Мой блог" по центру, текст тёмно-серый, фон белый
```

> 💡 **Правило:**
> Если можешь описать точнее — опиши. Лучше потратить 10 секунд на формулировку, чем 10 минут на переделку.

---

## Итерации

Не пытайся сделать идеально с первого раза. Уточняй:

1. **Ты:** Сделай кнопку «Отправить»

2. **Ты:** Сделай её синей

3. **Ты:** Добавь тень при наведении

Каждое сообщение уточняет предыдущее. Claude помнит весь разговор.

> ⚠️ **Важно:**
> Не начинай новый чат для каждого уточнения. Продолжай в том же — иначе Claude потеряет контекст.

---

## Примеры хороших запросов

**Создать:**
```
Создай страницу с формой: поле для имени и кнопка
```

**Изменить:**
```
Сделай текст крупнее
```

**Объяснить:**
```
Объясни что делает этот файл
```

**Найти:**
```
Найди где в проекте упоминается слово "пользователь"
```

---

## Чего избегать

**1. Слишком общие запросы** — «Сделай сайт» — Claude не знает какой.

**2. Много задач сразу** — лучше по одной, так проще контролировать.

**3. Думать что Claude «сам поймёт»** — он поймёт по-своему, и это может не совпасть.

---

## Контекст

Claude видит файлы в проекте. Используй это:

**Без контекста:** `Добавь кнопку` — куда? в какой файл?

**С контекстом:** `Посмотри index.html и добавь туда кнопку "Подписаться"` — точно знает что делать.

---

## Практика

1. **Создай файл:** `Создай файл hello.html с заголовком "Привет!"` — подтверди

2. **Добавь контент:** `Добавь абзац "Это моя первая страница"` — посмотри как Claude добавляет в существующий файл

3. **Измени:** `Сделай заголовок красным` — посмотри как работает контекст
