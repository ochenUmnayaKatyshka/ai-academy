# Цикл улучшения

У тебя есть документация. Но она не идеальна. Это нормально. Теперь — улучшаем.

---

## Цикл

```
Перечитать → «Ты уверен?» → Критический взгляд → Пересмотреть
```

Прогоняешь документацию через этот цикл 2–3 раза. После этого она будет сильно лучше.

---

## Шаг 1: Перечитать

Прочитай что написано. Как будто видишь впервые.

Вопросы к себе:

- Понятно ли человеку со стороны?

- Нет ли противоречий?

- Всё ли важное упомянуто?

---

## Шаг 2: «Ты уверен?»

Claude иногда уверенно говорит вещи, которые не совсем точны.

```
Ты упомянул что Telegram API бесплатный. Ты уверен? Есть ли ограничения?
```
```
Ты предложил Firebase. Какие альтернативы? Почему Firebase лучше?
```

Не надо проверять каждое слово. Но ключевые утверждения — да.

> ⚠️ **Важно:**
> Claude не врёт специально. Но может ошибаться в деталях. «Ты уверен?» — твой инструмент проверки.

---

## Шаг 3: Критический взгляд

Попроси Claude посмотреть критически:

```
Посмотри на документацию критически. Какие слабые места?
```
```
Представь что ты опытный разработчик. Какие вопросы возникнут?
```
```
Что я мог упустить?
```

Claude начнёт задавать неудобные вопросы. Это хорошо — лучше сейчас, чем когда уже сделана половина.

---

## Шаг 4: Пересмотреть

Обнови документацию:

```
Обнови: добавь раздел про обработку ошибок, убери уведомления — пока не нужно.
```

После изменений — снова шаг 1.

---

## Пример

**Было:**
```
## Основные функции
- Каталог товаров
- Корзина
- Оформление заказа
```

**После «Ты уверен?»:**

- Как товары добавляются? Вручную или из базы?

- Корзина сохраняется между сессиями?

- Что после оформления?

**После критического взгляда:**

- Нет обработки ошибок

- Нет уведомлений

- Нет админки

**Стало:**
```
## Основные функции
- Каталог товаров (редактируется через админку)
- Корзина (сохраняется в браузере)
- Оформление заказа (email-уведомление)
- Простая админка для товаров

## Обработка ситуаций
- Товар закончился → «Нет в наличии»
- Ошибка оплаты → сообщение + повтор
```

---

## Когда остановиться

2–3 прохода достаточно. Признаки:

- Новые правки мелкие

- Нет явных дыр

- По документации можно начать работу

> 💡 **Правило:**
> Документация достаточно хороша, когда по ней можно работать и не переспрашивать каждые 5 минут.

---

## Практика

Возьми документацию из прошлого урока:

1. Перечитай — что смущает?

2. «Ты уверен?» — проверь 2–3 утверждения

3. Критический взгляд — попроси Claude найти слабые места

4. Пересмотри — обнови
