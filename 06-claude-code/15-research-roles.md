# Исследование и роли

Claude может не только делать что скажешь, но и думать. Исследовать. Предлагать варианты. Это мощный инструмент, если знаешь как использовать.

---

## Режим исследования

Не всегда знаешь как сделать. Попроси Claude разобраться:

```
Какие есть способы сделать авторизацию? Сравни варианты.
```
```
Хочу добавить оплату на сайт. Какие варианты? Что проще?
```

Он найдёт информацию, сравнит, объяснит плюсы и минусы.

> 💡 **Совет:**
> Не принимай первый ответ как истину. Попроси сравнить несколько подходов.

---

## Роли

Попроси Claude посмотреть с определённой точки зрения:

```
Представь что ты опытный разработчик. Какие проблемы видишь?
```
```
Посмотри как тестировщик. Что может сломаться?
```

Роль меняет угол зрения. Claude начинает замечать то, что пропустил бы обычно.

---

## Переспрашивание

Ключевой навык:

```
Ты уверен что это безопасно?
```
```
Какие минусы у этого решения?
```
```
Что может пойти не так?
```

Переспрашивание заставляет Claude думать глубже.

> ⚠️ **Важно:**
> Особенно переспрашивай про безопасность. Если работаешь с данными пользователей — спрашивай «это безопасно?» и «какие уязвимости?».

---

## План перед действием

```
Хочу добавить форму регистрации. Предложи план в 3-5 шагов.
```

Claude напишет план. Ты можешь согласиться, изменить или спросить почему именно так.

---

## Критический анализ

После того как Claude что-то сделал:

```
Посмотри критически. Что можно улучшить?
```
```
Если бы это был чужой код — какие замечания?
```
