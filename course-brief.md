## Правила написания уроков

### 1. Объясняй каждый термин глубоко

Для каждого нового термина:
- **Что это** — определение простым языком
- **Зачем существует** — какую проблему решает
- **Роль в системе** — как связано с остальным
- **Аналогия из жизни** — из современного мира (Telegram, Uber, Netflix, Instagram). НЕ кулинария
- **Пример в коде** — с комментариями прямо внутри кода

Синтаксис объяснять комментариями внутри примера кода, не отдельным блоком.

### 2. Структура каждого урока

```
# Урок X.Y — Название

> Вводная цитата (1-2 предложения: зачем этот урок)

---

[Основное содержание: схемы, код, аналогии, пошаговые объяснения]

---

## Практика
### Задание 1-3 (руками сделать)

```блок-резюме всего урока```

Мостик к следующему уроку.
```

### 3. Визуальные схемы

ASCII-диаграммы для наглядности:
```
┌─────────────┐         ┌─────────────┐
│ Компонент A │ ──────► │ Компонент B │
└─────────────┘         └─────────────┘
```

### 4. Текст для внешних читателей

- Курс читают другие люди, НЕ автор
- Нельзя: «помнишь, мы настраивали», «ты уже видела»
- Можно: ссылки на предыдущие уроки («в уроке 2.3...»)
- Факты о продуктах — только из официальной документации

### 5. Источники

Для каждой ключевой концепции — ссылка:
```
Источник: Название
https://url
```

### 6. Не переусложняй

- Сначала ЗАЧЕМ, потом КАК
- Без «побочных плюсов» и «бонусов»
- Один новый концепт за раз
- Простое перед сложным
